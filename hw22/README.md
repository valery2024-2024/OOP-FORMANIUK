# Контрольні питання Лекції №22

## 1. У чому різниця між синхронним та асинхронним читанням файлів?

Синхронне читання блокує програму, поки файл читається.

Асинхронне читання не блокує програму і краще підходить для великих файлів або програм з інтерфейсом.

## 2. Які переваги має JSON над XML і навпаки?

JSON:

* менший за розміром;
* простіший;
* швидше читається;
* зручний для веб-додатків.

XML:

* підтримує коментарі;
* має XSD-схеми;
* краще підходить для enterprise-систем;
* використовується в SOAP і старих системах.

## 3. Як ігнорувати властивість при серіалізації в JSON та XML?

Для JSON:

```C#
[JsonIgnore]
public string InternalCode { get; set; }
```

Для XML:

```C#
[XmlIgnore]
public string InternalCode { get; set; }
```

## 4. Чому важливо використовувати using при роботі з потоками?

* Бо using автоматично закриває файл після роботи.

* Це захищає від помилок і блокування файлів.

## 5. Як обробляти помилки при десеріалізації некоректного JSON?

Потрібно використовувати try-catch і ловити JsonException.

```C#
try
{
    var product = JsonSerializer.Deserialize<Product>(json);
}
catch (JsonException)
{
    Console.WriteLine("JSON має неправильний формат");
}
```
