# **Домашня робота по лекції №17**

## **Породжувальні патерни: `Factory Method і Singleton`**

## **Тема лекції**

Породжувальні патерни проєктування: **`Factory Method`** та **`Singleton`**.

## **Мета лекції**

Зрозуміти, що таке патерни проєктування, навіщо вони використовуються, та розібрати принцип роботи породжувальних патернів Factory Method і Singleton.

## 1.**Патерни проєктування**

Це перевірені підходи до розв’язання типових проблем у програмуванні. Вони не є готовим кодом, а описують загальну ідею, яку можна реалізувати у власному проєкті.

Патерни допомагають:

- повторно використовувати перевірені рішення;
- писати більш зрозумілий і підтримуваний код;
- спростити взаєморозуміння між програмістами.

За призначенням патерни поділяються на:

- **породжувальні** - відповідають за створення об’єктів;
- **структурні** - визначають будову класів і об’єктів;
- **поведінкові** - описують взаємодію між об’єктами.

У даній лекції розглядаються саме породжувальні патерни.

## 2. Патерн **`Factory Method`**

Це патерн, який визначає інтерфейс для створення об’єкта, але дозволяє підкласам вирішувати, який саме об’єкт створювати.

Цей патерн використовується тоді, коли клієнтський код не повинен знати, який конкретний клас створюється. Клієнт працює лише з інтерфейсом, а вибір реалізації делегується фабриці.

### **Основні елементи**

- **`Product`** — інтерфейс або абстрактний клас;
- **`ConcreteProduct`** — конкретна реалізація продукту;
- **`Creator`** — абстрактна фабрика з фабричним методом;
- **`ConcreteCreator`** — конкретна фабрика, яка створює продукт.

У прикладі з логером фабрика може створювати консольний або файловий логер. При додаванні нового типу логера клієнтський код не змінюється.

### **Переваги `Factory Method`**

- слабка залежність між класами;
- легке розширення системи;
- дотримання принципу відкритості/закритості (OCP).

## 3. Патерн **Singleton**

Це патерн, який гарантує, що клас має лише один екземпляр і надає глобальну точку доступу до нього.

`Singleton` застосовується для класів, об’єкт яких повинен бути єдиним у системі, наприклад: менеджер конфігурації або логер.

### **Основні ознаки**

- приватний конструктор, який забороняє створення об’єкта через `new`;
- приватне статичне поле для зберігання екземпляра;
- публічна статична властивість або метод для доступу до цього екземпляра.

#### **Переваги:**

- контрольований доступ до ресурсу;
- економія пам’яті та ресурсів.

#### **Недоліки:**

- складніше тестувати код;
- можливе порушення принципів SOLID при зловживанні патерном.

## 4. **Factory Method vs Abstract Factory**

**Factory Method**
Використовується для створення одного типу об’єкта з різними реалізаціями. Він простіший у реалізації та підходить для відносно простих задач.

**Abstract Factory**
Застосовується тоді, коли потрібно створювати набори взаємопов’язаних об’єктів (сімейства об’єктів). Цей патерн складніший, але забезпечує узгодженість створених компонентів.

### **Вибір патерна**

- `Factory Method` - для створення одного об’єкта різними способами;
- `Abstract Factory` - для створення групи пов’язаних об’єктів.

## **Висновок**

Патерни `Factory Method і Singleton` дозволяють створювати гнучкі та зручні для розширення програми. Їх використання зменшує залежність між класами та покращує загальну архітектуру програмного забезпечення.
